Design: Agentic Auditor Financial RAG

A high-fidelity system designed for numeric accuracy and narrative correctness.

1. Data Ingestion & Extraction

To ensure "perfect" extraction from unstructured PDFs:

  • Parser: Use Marker or LlamaParse to convert PDFs into high-quality Markdown. Markdown preserves structural cues (bolding, headers) that are vital for financial context.
  • Chunking: RecursiveCharacterTextSplitter with Markdown headers as delimiters.
  • Metadata: Each chunk tagged with source_file, page_number, and document_type.

2. Vector Storage & Retrieval

  • Embedding: text-embedding-3-small (OpenAI) or bge-large-en-v1.5 for high semantic density.
  • Vector DB: ChromaDB or Qdrant (local-first, scalable to API).
  • Hybrid Search: Combine semantic search with BM25 keyword matching. This ensures that specific financial terms or unique IDs are found even if semantic similarity is low.

3. Generation & Verification (The "Perfect Answer" Loop)

Agentic Workflow
Agent A: Extractor
Synthesizes answer from chunks.
→
Agent B: Auditor
Verifies every amount against raw chunks.

If Agent B finds a discrepancy (e.g., "$1.2M" in answer vs "$1.2B" in source), it triggers a regeneration with a corrective prompt.
