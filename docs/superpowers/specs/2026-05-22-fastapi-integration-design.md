# Design Spec: FastAPI Integration for Financial RAG

This document outlines the design for adding a FastAPI-based REST API to the Financial RAG project.

## Goal
Provide a synchronous API endpoint that allows users to submit queries, automatically retrieves context from the vector store, and returns an answer generated by the LangGraph agent workflow.

## Architecture

### Components
1.  **FastAPI Application (`src/api.py`)**: The entry point for the web service.
2.  **Vector Store Integration**: Uses the existing `Ingestor.search` method to find relevant context.
3.  **Agent Workflow Integration**: Uses the existing `run_financial_rag` function to process the query with the retrieved context.

### Endpoints
*   **`GET /health`**: Simple health check to verify the API is running.
*   **`POST /query`**: The primary RAG endpoint.
    *   **Input**: `QueryRequest` (JSON)
    *   **Logic**:
        1.  Search ChromaDB for relevant documents based on the input query.
        2.  Format the retrieved documents into a single context string.
        3.  Execute the LangGraph workflow with the query and context.
    *   **Output**: `QueryResponse` (JSON)

## Data Models (Pydantic)

### `QueryRequest`
```json
{
  "query": "What is the total revenue?"
}
```

### `QueryResponse`
```json
{
  "query": "What is the total revenue?",
  "answer": "The total revenue was $1.2 billion.",
  "sources": [
    { "content": "...", "metadata": { "page": 1 } }
  ]
}
```

## Implementation Plan

1.  **Environment Setup**: Add `fastapi` and `uvicorn` to `requirements.txt`.
2.  **API Development**: Create `src/api.py` with the defined endpoints and logic.
3.  **Refactoring (if needed)**: Ensure `Ingestor` and `run_financial_rag` can be easily imported and initialized without side effects.
4.  **Testing**: Create `tests/test_api.py` using FastAPI's `TestClient` to verify the integration end-to-end (mocking agent calls as necessary).

## Success Criteria
*   The API starts successfully with `uvicorn`.
*   A POST request to `/query` returns a valid answer and source documents.
*   All existing tests continue to pass.
